2. STAC Browser: Understanding STAC

  • Definition: SpatioTemporal Asset Catalog - the "HTML of geospatial data"
  • Purpose: Standardized way to describe and discover Earth observation data
  • Benefits: Interoperability, discoverability, efficiency, scalability
  • Problem Solved: Unified integration across multiple data providers
Search any STAC catalog with same syntax:
catalog.search(bbox=[-180, -90, 180, 90], datetime="2024-01")

Guide: STAC/stac-training-guide.md

2. STAC Browser: Hierarchy & Components

  • Catalog: Top-level entry point to dataset
  • Collection: Groups of related items (e.g., Landsat-8)
  • Item: Single spatiotemporal dataset (satellite scene)
  • Assets: Actual files (data, metadata, thumbnails)
Catalog
├── Collection (Landsat-8)
│   ├── Item (Scene from 2024-01-01)
│   │   ├── Asset (Band 1)
│   │   └── Asset (Thumbnail)

Reference: STAC/stac-training-guide.md

2. STAC Browser: Working with VEDA STAC

  • API Endpoint: https://openveda.cloud/api/stac/
  • Collections: NASA disasters, air quality, climate data
  • Search: Spatial, temporal, and property-based queries
  • Python Access: Using pystac-client library
from pystac_client import Client
veda_stac = Client.open("https://openveda.cloud/api/stac/")
search = veda_stac.search(collections=["air-quality-no2"])

Full guide: STAC/stac-training-guide.md

5. Data Ingest: COG Transformation Pipeline

  • COG: Cloud Optimized GeoTIFF format
  • Process: Raw data → Transform → COG → S3
  • Compression: DEFLATE with appropriate predictors
  • Validation: rio cogeo validate file.tif
COG_PROFILE = {
    "driver": "COG",
    "compress": "DEFLATE",
    "blocksize": 512
}

Examples: Data-ingest/cog-conversions.md
